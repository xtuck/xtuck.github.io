LAN Share Pro - Technical Support

Help for setup, LAN access, upload/download, and troubleshooting.

Updated: 2026-03-09

1. Contact Support

If you need help, contact us at: gxtuna@gmail.com

Please include your app version, macOS version, and a short problem description.

2. Before You Contact Us

  • Confirm the service is running in the app Home page.
  • Confirm both devices are in the same local network (same LAN).
  • Use the displayed local URL (http://IP:PORT) or scan the QR code.
  • If temporary passcode is enabled, log in first.
  • Check app Logs for errors and include key lines in your email.

3. Common Issues

Cannot open the web page from another device

  • Verify you are using the exact IP and port shown in the app.
  • Check local firewall settings on the host Mac.
  • Try refreshing the service status or restarting the service.

Upload failed

  • Check available cache space and configured cache limit.
  • Reduce concurrent uploads in Settings if needed.
  • Check whether file names/paths contain unsupported patterns.

Download is slow or blocked

  • Check current download concurrency limits.
  • Try another browser on the same LAN.
  • Check network quality between the two devices.

4. HTTPS and Certificate Notes

  • The app supports optional HTTPS with a self-signed certificate.
  • Client devices may need to trust the certificate manually.
  • If HTTPS access fails, test HTTP first to isolate certificate issues.

5. API Support

LAN Share Pro supports HTTP upload/download APIs for automation (for example, with curl). For endpoint or usage issues, include your request sample and response status code when contacting support.

6. Security Reminder

  • Use temporary passcode protection when sharing in multi-user networks.
  • Avoid exposing your local service to the public internet unless you understand the risks.
  • Only share with trusted devices and networks.

7. Feedback

Feature requests and bug reports are welcome at gxtuna@gmail.com.
